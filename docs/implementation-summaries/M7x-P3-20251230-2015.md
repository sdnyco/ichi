# M7x P3 — place_context first-load race fix

- `getOrCreatePlaceProfile` now uses a single `insert … on conflict … returning` with the correct `(user_id, place_id)` target, so concurrent requests (e.g. double fetches from Safari hydration) no longer hit the unique constraint.
- `PlacePingPanel` guards `/api/me/place-context` fetches with an `AbortController`, deduplicates requests per `{placeId,userId}`, and logs status/body snippets when failures occur. This prevents overlapping calls and gives actionable logs if the API ever fails again.
